# Skill: 要件定義 (Requirement Expert)

## 概要
ユーザーとの対話を通じて「何を作りたいか」を詳細に定義し、GitHub Issue を唯一の真実のソース（SSOT）として更新する。

## Instructions
1. **Existing Issue Deep Dive**: 既存のIssueの内容を分析し、不足している情報をインタビューで補完する。
2. **New Issue Definition**: 新しい機能や修正の要望をゼロからインタビューし、構造化する。
3. **SSOT 更新**: 全ての情報を統合し、GitHub Issue の Body を最新の要件で更新する。

## Output Contract
- GitHub Issue の最新化された Body (Background, Success Criteria, etc.)
- ユーザー合意が得られた要件定義
