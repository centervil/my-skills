# GitHub Issue Label Guidelines

このプロジェクトでは、Issueの種類を明確にし、フィルタリングを容易にするために以下の基準でラベルを付与します。

| ラベル | 判定基準 | 優先度 |
| :--- | :--- | :--- |
| `bug` | 予期しない挙動、エラー、クラッシュ、セキュリティ上の欠陥。 | 高 |
| `enhancement` | 既存機能の改善、パフォーマンス向上、リファクタリング（機能変更を伴う場合）。 | 中 |
| `feature` | 全く新しい機能の追加要件。 | 中 |
| `documentation` | README、Wiki、コード内コメント、外部ドキュメントの更新。 | 低 |
| `chore` | ビルド設定、依存関係の更新、CI/CDの設定変更など、コードの論理に影響しない作業。 | 低 |
| `question` | 仕様の確認、実装方針の相談、バグかどうかの判断が必要な事項。 | 中 |
| `design` | UI/UXのデザイン変更、CSSの調整など。 | 中 |

## 運用のルール
1. **最小限の付与**: 1つのIssueにつき、主要なラベルを1つ選択します（例: `bug` と `feature` は同時には付与しない）。
2. **迷った場合**: 判断がつかない場合は `question` を付与し、本文の冒頭に「要判断」と記載してください。
3. **重複の禁止**: すでに同様のIssueが存在しないか、作成前に必ず検索してください。
